💾 ALMACENAMIENTO
═══════════════════════════════════════════════════════════════════════════════

MongoDB Collection: f1caches
├── endpoint: String (unique, indexed)
├── data: JSON (cualquier estructura)
├── lastUpdated: Date
└── expiresAt: Date (TTL index = auto-cleanup)

Ventaja vs archivos locales:
✅ Acceso instantáneo
✅ Query y filtrado flexible
✅ Escalable (millones de docs)
✅ TTL automático (auto-cleanup)
✅ Replicación y backup fácil

🎯 CASOS DE USO
═══════════════════════════════════════════════════════════════════════════════

1. Página de Pilotos
   → useF1Data({ endpoint: 'drivers' })
   → Lista de pilotos se carga al instante (desde caché)

2. Dashboard en Vivo (fin de semana)
   → useF1Data({ endpoint: 'laps', queryParams: { session_key: 9158 }, refetchInterval: 5000 })
   → Se actualiza cada 5 segundos (pero sirve desde caché)

3. Estadísticas Históricas
   → GET /api/f1/races?year=2024
   → Todos los datos con un solo endpoint

4. App Móvil
   → Mismos endpoints, reduce de 10s a <100ms por fetch
